Sankore API Documentation

This documentation describes the Sankore API. This API is useable in the widgets through the 'sankore' object.
For more information, have a look at the example widget example.wgt.

Here is an example of the way the method returnStatus is called from the widget:

function startEditing()
{
	sankore.returnStatus("startEditing()", "Ok!");
}

This code sample is the declaration of the method Start Edit that can be called from Sankore. This
method must be declared in the widget and implemented to do the proper action. In this example, this method
will display the debug message "startEditing():Ok!" in Sankore.

Sankore to Widget

startEditing

Description

Put the widget in edit mode.

stopEditing

Description

Put the widget in display mode.

initialize

Description

Put the widget in a known state (initial state).

checkResponse

Description

Ask the widget for its response.

reload

Description

Reload the widget.

save

Description

Save the widget state.

listMethods

Description

Ask the widget which method of the Sankore API are used. This method is used to generate the widget buttons.

Widget to Sankore

returnStatus

Parameters

[QString] method: the called method.
[QString] status: the actual status.

Description

Return the status of the widget. This method is used for debug purpose only.

usedMethods

Parameters

[QStringList] methods: the used API methods list.

Description

Return the used API methods of the widget.

response

Parameters

[bool] correct: the response state.

Description

Return an indication of the rightness of the widget's response.

locale

Return value

[QString] the actual language used by the Sankore application.

Description

Return the actual language identifier used by the Sankore application.
